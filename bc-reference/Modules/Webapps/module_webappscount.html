{module_webappscount}

Description

Count of items in the individual Web App.

This module also supports Liquid, take a look at this article to learn how to access its Liquid output tags and render it using your own custom logic.

  • New named parameters documentation
  • Legacy module documentation

Syntax

{% raw %}{module_webappscount id="webappId|webappName" filter="all" itemId="categoryId|categoryName" collection="my_custom_collection_name" template=""}{% endraw %}

Note: When using named parameters always wrap the individual parameters in double quotes. Also, do note that modules using named parameters do not work for campaigns just yet.

Parameters

  • id - the web app name or the web app's ID
  • filter - filtering criteria for display and can be one of the following:
    • all - all items
    • classified - all the items in a category
  • itemId - the category ID or the category name
  • collection - use this if you need to access the module's Liquid output tags
  • template - this module does not support custom templates. You can use the template parameter however to supress the module's output

Examples

  • {% raw %}{module_webappscount id="56"}{% endraw %} - assuming there are 4 webapp items in the webapp with the ID 56 this module will render "4"
  • {% raw %}{module_webappscount id="56" collection="counter" template=""}{% endraw %} - assuming there are 4 webapp items in the webapp with the ID 56 the Liquid code {% raw %}{{this.counter.count | json}}{% endraw %} will render "4"

Syntax

{% raw %}{module_webappscount,ID}{% endraw %}
  • ID - system generated web app id (do not change)

Examples

  • {% raw %}{module_webappscount,56}{% endraw %} - assuming there are 4 webapp items in the webapp with the ID 56 this module will render "4"
