Webapp detail layout

Is used by these modules

Site visitors get to it only after clicking the links outputted by the tag {% raw %}{tag_name}{% endraw %} or {% raw %}{tag_button}{% endraw %} which can both be found on the list layout.

Accessible from

  • Admin Console: Site Manager > Module Templates >Web App Layouts > WEBAPP_NAME
  • SFTP & Develop Mode: /Layouts/WebApps/WEBAPP_NAME/detail.html (for example /Layouts/WebApps/Car Catalogue/detail.html)

Renders these tags

Warning: if you need to access a liquid tag that has a space in its name, for example tag_large image_value use this syntax: {% raw %}{{this.items[0].["large image_value"]}}{% endraw %} - Liquid does not allow for the name of its output tags to contain spaces. In this case the tag that contains a space needs to be wrapped in square brackets and quotes and preceded by "this.".

Note: When using a custom image field type for Web Apps, by default the image tag outputs the full HTML image source and a link. Example: 

{% raw %}<a href="/web-app/detail-web-app"><img src="/folder/imagename.jpg" border="0" alt=""></a>{% endraw %}

You can customize custom image tags, by appending the below parameters to the tags:

  • _value eg. {% raw %}{tag_my custom image_value}{% endraw %} - Outputs the file path of the image. For example {% raw %}/folder/image.jpg{% endraw %}
  • _nolink eg. {% raw %}{tag_my custom image_nolink}{% endraw %} - Ensures that the image does not link to the detailed view of the web app item
