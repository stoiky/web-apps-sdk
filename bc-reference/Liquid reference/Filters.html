Mathematical Filters

{% raw %}divided_by{% endraw %}

division - {% raw %}{{500 | divided_by: 20}}{% endraw %} is 25
Note: when you divide a number by another number which has one decimal the result will also have an output:
{% raw %}{{399 | divided_by: 2.0}} {% endraw %} will output: 199.5

 
{% raw %}minus{% endraw %} substraction - {% raw %}{{500 | minus: 20}}{% endraw %} is 480
{% raw %}plus{% endraw %} addition - {% raw %}{{500 | plus: 20}}{% endraw %} is 520
{% raw %}times{% endraw %} multiplication - {% raw %}{{500 | times: 2}}{% endraw %} is 1000
{% raw %}modulo{% endraw %} the remainder - {% raw %}{{3 | modulo: 2}}{% endraw %} is 1, {% raw %}{{6 | modulo: 2}}{% endraw %} is 0
{% raw %}floor{% endraw %} rounded down - {% raw %}{{4.6|floor}} {% endraw %} is 4
{% raw %}ceil{% endraw %} rounded up - {% raw %}{{4.3|ceil}} {% endraw %} is 5
{% raw %}round{% endraw %} typical rounding - {% raw %}{{4.5|round}} {% endraw %} is 5 &{% raw %}{{4.4|round}} {% endraw %} is 4
{% raw %}round:x{% endraw %} rounding by number of decimals - {% raw %}{{4.5612|round:2}} {% endraw %} is 4.56

Array manipulation filters

{% raw %}limit{% endraw %} array = [1,2,3,4,5,6]
{% raw %}{% for item in array limit:2 %}{% endraw %}
This will limit the number of items returned from the array. In this case 1 and 2 will be returned
{% raw %}offset{% endraw %} array = [1,2,3,4,5,6]
{% raw %}{% for item in array offset:2 %}{% endraw %}
This will offset the items from the array. In this case items 3 and 4 will be returned.
You can combine both offset and limit.
{% raw %}join{% endraw %} {% raw %}{{myArray | join: "///"}}{% endraw %} assuming myArray is an array joins the elements and puts the separator between each element
{% raw %}sort{% endraw %} {% raw %}{{myArray | sort}}{% endraw %} assuming myArray is an array ["5","4","3"] ]the sort filter outputs ["3","4","5"]
{% raw %}first{% endraw %} {% raw %}{{myArray | first}}{% endraw %} Will output the first item in an array
{% raw %}last{% endraw %} {% raw %}{{myArray | last}}{% endraw %} Will output the last item in an array

Date switches

Assuming the value of the {% raw %}dateTime{% endraw %} field is 2014-12-09T23:00:00 here are the available switches you can use to format the date:

{% raw %}%d{% endraw %} Outputs the day of the month - 18
{% raw %}dd{% endraw %} The day of the month. Single-digit days will have a leading zero - 03
{% raw %}ddd{% endraw %} The abbreviated name of the day of the week. - Tue
{% raw %}dddd{% endraw %} The full name of the day of the week. - Tuesday
{% raw %}%h{% endraw %} The hour in a 12-hour clock. Single-digit hours will not have a leading zero - 12
{% raw %}hh{% endraw %} The hour in a 12-hour clock. Single-digit hours will have a leading zero - 12
{% raw %}%H{% endraw %} The hour in a 24-hour clock. Single-digit hours will not have a leading zero - 0
{% raw %}HH{% endraw %} The hour in a 24-hour clock. Single-digit hours will have a leading zero - 0
{% raw %}%m{% endraw %} The minute. Single-digit minutes will not have a leading zero - 0
{% raw %}mm{% endraw %} The minute. Single-digit minutes will have a leading zero - 00
{% raw %}%M{% endraw %} The numeric month. Single-digit months will not have a leading zero - 9
{% raw %}MM{% endraw %} The numeric month. Single-digit months will have a leading zero - 09
{% raw %}MMM{% endraw %} The abbreviated name of the month, as defined in AbbreviatedMonthNames - sep.
{% raw %}MMMM{% endraw %} The full name of the month, as defined in MonthNames - september
{% raw %}%s{% endraw %} The second. Single-digit seconds will not have a leading zeroThe second. Single-digit seconds will not have a leading zero - 0
{% raw %}ss{% endraw %} The second. Single-digit seconds will have a leading zero - 00
{% raw %}%t{% endraw %} The first character in the AM/PM designator, if any - a
{% raw %}tt{% endraw %} The AM/PM designator, if any - a.m.
{% raw %}%y{% endraw %} The year without the century. If the year without the century is less than 10, the year is displayed with no leading zero - 14
{% raw %}yy{% endraw %} The year without the century. If the year without the century is less than 10, the year is displayed with a leading zero - 14
{% raw %}yyyy{% endraw %} The year in four or five digits (depending on the calendar used), including the century. Will pad with leading zeroes to get four digits. Thai Buddhist and Korean calendars both have five digit years; users selecting the "yyyy" pattern will see all five digits without leading zeros for calendars that have five digits. Exception: the Japanese and Taiwan calendars always behave as if "yy" was selected - 1914
{% raw %}\c{% endraw %} Where c is any character. Displays the character literally. Useful for escaping characters that are usually rendered as switches (m, h, s, etc.) To display the backslash character, use "\\" - 14

To output 14-6-18 use {% raw %}{{dateVariable | date: "%y-%M-%d"}}{% endraw %} (mind the double quotes).

To output 1914\09\03 @ 12:00:00 a.m. use {% raw %}{{dateVariable | date: "yyyy\\MM\\dd @ hh:mm:ss tt"}}{% endraw %}

To use "\" (the backslash character) as separator for date elements you need to escape the it in the liquid tag. Using "\\" renders "\" in the front-end.

Spaces and any other characters that are not usable switches are rendered without the need for escaping.

timeSpan switches

Let's assume we have created a timeSpan variable by substracting one date from another:

 {% raw %}{% assign dateDiff = globals.site.dateNow | minus: myDate %}{% endraw %}

Taking a presumed value of dateDiff being 5.02:39:20.2500250, we can format the date value of {% raw %}{{dateDiff}}{% endraw %} with the following switches:

{% raw %}%d{% endraw %} Outputs the number of whole days in the time interval - 5
{% raw %}dd{% endraw %} The number of whole days in the time interval, padded with leading zeros as needed - 05
{% raw %}%h{% endraw %} The number of whole hours in the time interval that are not counted as part of days. Single-digit hours do not have a leading zero - 2
{% raw %}hh{% endraw %} The number of whole hours in the time interval that are not counted as part of days. Single-digit hours have a leading zero - 02
{% raw %}%m{% endraw %} The number of whole minutes in the time interval that are not included as part of hours or days. Single-digit minutes do not have a leading zero - 39
{% raw %}mm{% endraw %} The number of whole minutes in the time interval that are not included as part of hours or days. Single-digit minutes have a leading zero - 39
{% raw %}%s{% endraw %} The number of whole seconds in the time interval that are not included as part of hours, days, or minutes. Single-digit seconds do not have a leading zero - 20
{% raw %}ss{% endraw %} The number of whole seconds in the time interval that are not included as part of hours, days, or minutes. Single-digit seconds have a leading zero - 20
{% raw %}%f{% endraw %} The tenths of a second in a time interval - 2
{% raw %}ff{% endraw %} The /hundredths of a second in a time interval - 25
{% raw %}fff{% endraw %} The milliseconds in a time interval - 250
{% raw %}ffff{% endraw %} The ten-thousandths of a second in a time interval - 2500
{% raw %}fffff{% endraw %} The hundred-thousandths of a second in a time interval - 25002
{% raw %}ffffff{% endraw %} The millionths of a second in a time interval - 250025
{% raw %}fffffff{% endraw %} The ten-millionths of a second (or the fractional ticks) in a time interval - 2500250
{% raw %}%F{% endraw %} The tenths of a second in a time interval. Nothing is displayed if the digit is zero -2
{% raw %}FF{% endraw %} The hundredths of a second in a time interval. Any fractional trailing zeros or two zero digits are not included - 25
{% raw %}FFF{% endraw %} The milliseconds in a time interval. Any fractional trailing zeros are not included - 25
{% raw %}FFFF{% endraw %} The ten-thousandths of a second in a time interval. Any fractional trailing zeros are not included - 25
{% raw %}FFFFF{% endraw %} The hundred-thousandths of a second in a time interval. Any fractional trailing zeros are not included - 25002
{% raw %}FFFFFF{% endraw %} The millionths of a second in a time interval. Any fractional trailing zeros are not displayed - 250025
{% raw %}FFFFFFF{% endraw %} The ten-millions of a second in a time interval. Any fractional trailing zeros or seven zeros are not displayed - 250025
{% raw %}string{% endraw %} Literal string delimiter - date:"'Result : 'dd'"'' will output - Result : 05
\ The escape character. If you want to display a cahracter that would be normally interpreted as a switch, for example /ddd will output - d05

To output 5-3-13 use {{dateDiff | date: "%d'-'%h'-'%m"}} (mind the double quotes).

To output Result : d05-h03-m18 use {% raw %}{{dateDiff | date:"'Result : '\ddd'-'\hhh'-'\mmm"}}{% endraw %}

Other Filters

{% raw %}convert{% endraw %} Converts variables into different formats. Possible dataTypes are:
  • string - {% raw %}{{3 | convert: "string" | json}}{% endraw %} outputs "3"
  • number - {% raw %}{{"3" | convert: "number" | json}}{% endraw %} outputs 3 (this can now be used with the mathematical filters)
  • boolean - {% raw %}{{"false" | convert: "boolean" | json}}{% endraw %} outputs "false" (it is treated as a boolean variable)
  • date - {% raw %}{{"2015-01-07" | convert: "date"}}{% endraw %} outputs 2015-01-07T00:00:00 in dateTime format
  • currency(money) {% raw %}{{100 | convert: "money"}}{% endraw %} outputs "100 USD" assuming the site's "Country code" is set to US.
{% raw %}json{% endraw %} Displays the data passed in JSON format. For example rendering a single webapp item using - {% raw %}{{webapp1.items[0] | json}}{% endraw %} looks like: 
{
 "itemid":"4646524",
 "name":"ABA",
 "urlWithHost":"http://abcd.com/car/ABA",
 "url":"/car/ABA",
 "releaseDate":"2014-06-18",
 "expiryDate":"9999-01-01",
 "lastUpdateDate":"2014-06-18",
 "counter":"1",
 "weight":"",
 "Make":"AC model",
 "Color":"red",
 "Country":"",
 "Review":"no review just yet"
}

Do note you need to make the webapp listing available in a collection before you can access the items in JSON format.
{% raw %}size{% endraw %} Can be used to return the number of elements in an array or the number of characters in a string.
{% raw %}unescape{% endraw %} Opposite of escape, basically HTML decode.{% raw %} {{"<div style='color:red'><h1>header</h1><p>this is a paragraph</p></div>" | unescape}}{% endraw %}
{% raw %}url_param_escape/ url_param_unescape {% endraw %} Encodes/decodes a parameter than can be appended to an URL.{% raw %} {{"<div> & <span>" | url_param_escape}}{% endraw %} / {% raw %} {{"&lt;hello&gt; &amp; &lt;shopify&gt;" | url_param_unescape}}{% endraw %}
{% raw %}money{% endraw %} {% raw %} {{ 145.9 | money }} {% endraw %} Formats the price based on the associated currency. (USD145.9)
{% raw %}default{% endraw %} {% raw %} Dear {{ customer.name | default: "customer" }} {% endraw %} Sets a default value for any variable with no assigned value. Can be used with strings, arrays, and hashes
