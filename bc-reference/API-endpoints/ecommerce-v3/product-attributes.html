Product Attributes

    List all the product's attributes of a site

    Request

    • Method: GET
    • Version: v3, this API call supports additional sort and filter mechanisms as well as retrieving only a sub-set of fields

      Note: In order to access the v3 API endpoints please make sure you have enabled the new rendering engine.

    • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
      • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
    • Path: /webresources/api/v3/sites/current/productsattributes
      • Alternatively, use siteID instead of 'current'
    • Authorization header: This should contain the authorization token. Here is how to obtain the token.

    Response

    An array of the first 10 attributes; each object has the following properties:

    • id - the attribute's ID (numeric)
    • siteId - the site's ID (numeric)
    • productId - the product's ID (numeric)
    • groupName - the attribute's name(string)
    • fieldTypeId - the attribute's display type (integer)
      • 5 - Drop Down List
      • 6 - Check Box List
      • 7 - Radio List
    • mandatory - whether this attribute is required when adding the product to cart (boolean)
    • keepStock - whether this attribute is used to track stock (boolean)

    You can output additional fields to those listed above as default, see the API v3 Discovery APP for additional details.

    Accepts and returns JSON as Content-Type.

    Create a product's attribute

    Request

    • Method: POST
    • Version: v3, this API call supports additional sort and filter mechanisms as well as retrieving only a sub-set of fields

      Note: In order to access the v3 API endpoints please make sure you have enabled the new rendering engine.

    • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
      • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
    • Path: /webresources/api/v3/sites/current/productsattributes
      • Alternatively, use siteID instead of 'current'
    • Authorization header: This should contain the authorization token. Here is how to obtain the token.

    Parameters

    • productId - the product's ID, required (numeric)
    • groupName - the attribute's name(string)
    • fieldTypeId - the attribute's display type (integer)
      • 5 - Drop Down List
      • 6 - Check Box List
      • 7 - Radio List
    • mandatory - whether this attribute is required when adding the product to cart (boolean)
    • keepStock - whether this attribute is used to track stock (boolean)
      • can't be set true when fieldTypeId is 6
      • can't be set true when mandatory field is false

    Response:

    {% raw %}HTTP/1.1 201 CREATED
Location: /webresources/api/v3/sites/current/productsattributes
Content-Type: application/json; charset=utf-8
Content-Length: length{% endraw %}

        "Authorization": $.cookie('access_token')
    },
    processData: false,
    data: JSON.stringify({
        "productId": 951894,

    })
});

    console.log(msg);
